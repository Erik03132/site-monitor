Этот воркфлоу — священный ритуал начала проекта. Он призывает "команду" (Архитектора, Разработчика и QA) и закладывает незыблемый фундамент кода.

### Шаг 1: Сбор Команды (Настройка .cursorrules)
Необходимо создать или обновить файл `.cursorrules`, чтобы агент знал свои роли.
// turbo
1. Создать `.cursorrules` с определением ролей: Архитектор (контроль структуры), Senior Developer (чистый код), QA Engineer (тесты).

### Шаг 2: Подбор Навыков системы
Используйте навык `using-superpowers`, чтобы найти и активировать лучшие практики для текущего стека.
2. Выполнить поиск навыков по ключевым словам проекта (напр. `nextjs`, `supabase`, `python`).

### Шаг 3: Инициализация Кондуктора
Подключение системы управления проектом Google Conductor.
// turbo
3. Запустить команду `conductor-setup` для фиксации Product Definition и Tech Stack.

### Шаг 4: Нулевой аудит
Создание первого проверочного трека.
4. Запустить `conductor-new-track` с названием "Initial System Audit" для проверки базовой работоспособности и отсутствия CVE.

### Шаг 5: Фиксация правил
Добавление специфических правил проекта (например, "Писать только на русском", "Использовать только Tailwind v4").
5. Добавить эти правила в раздел `user_rules` в `.cursorrules`.
