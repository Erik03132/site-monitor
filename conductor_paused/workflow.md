# Workflow: Aether Monitor

## Принципы разработки
1. **TDD (Test-Driven Development)**: Тесты поощряются для сложной логики парсинга и диффа.
2. **Commit Strategy**: Conventional Commits (`feat:`, `fix:`, `chore:`).
3. **Atomic Changes**: Одна задача — один коммит/PR.
4. **Verfication**: Ручная проверка работоспособности после каждого этапа (Phase).

## Жизненный цикл задачи (Track)
1. **Spec**: Определение требований.
2. **Plan**: Пошаговый план реализации.
3. **Implementation**: Написание кода.
4. **Verification**: Тестирование.
5. **Closure**: Слияние и архивация.
